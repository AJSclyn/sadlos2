Introduction to SADL 3

Last revised 12/07/2016. Send comments to mailto:crapo@research.ge.com.

SADL 3 is a reimplementation of the Semantic Application Design Language using Xtext 2.9+. The grammar is a complete reimplementation but is largely backward compatible with SADL 2. For non-backward compatibilities, see What's Changed in the Grammar.

The purpose of the reimplementation is to have a cleaner grammar that does not require parser backtracking so that content assist and other Xtext-based IDE features are more robust and useful with fewer customizations. In addition, SADL 3 uses Xtext-based scoping and linking instead of using the Jena ontology model interface. This allows inter-project dependencies and more robust and scaleable linking of both concepts and imported models. It also means that SADL is less dependent upon Jena and non-Jena-based SADL model processors could be created, opening up the possibility of support for OWL 2. For a complete list of changes see What's New in the IDE.

What's New in the Grammar

  1. Typed lists are supported with an OWL implementation
    • Example

    Person is a class described by children with values of type Person List.

    • What's happening behind the scenes-- here is the SADL typed list meta-model in SADL syntax:

    ^List is a class
                described by ^first,
                described by rest with values of type ^List,
                described by lengthRestriction with values of type int,
                described by minLengthRestriction with values of type int,
                described by maxLengthRestriction with values of type int.

    Here is part of the corresponding OWL model for the example above:

    <owl:ObjectProperty rdf:ID="children">
            <rdfs:domain rdf:resource="#Person"/>
            <rdfs:range rdf:nodeID="A0"/>
    </owl:ObjectProperty>
    <owl:Class rdf:nodeID="A0">
            <rdfs:subClassOf rdf:resource="sadllistmodel#List"/>
            <rdfs:subClassOf>
                    <owl:Restriction>
                            <owl:allValuesFrom>
                                    <owl:Class rdf:ID="Person"/>
                            </owl:allValuesFrom>
                            <owl:onProperty rdf:resource="sadllistmodel#first"/>
                    </owl:Restriction>
            </rdfs:subClassOf>
            <rdfs:subClassOf>
                    <owl:Restriction>
                            <owl:allValuesFrom rdf:nodeID="A0"/>
                            <owl:onProperty rdf:resource="sadllistmodel#rest"/>
                    </owl:Restriction>
            </rdfs:subClassOf>
    </owl:Class>
  2. List operations include:
    • "length of <list>"
    • "count of <element> in <list>"
    • "index of <element> in <list>"
    • "the sublist of <list> matching <conditions>"
    • "<list> contains <element>", "<list> does not contain <element>"
    • "<element> is unique in <list>", "<element> is not unique in <list>"
    • "element before <element> in <list>", "element after <element> in <list>"
    • "first element of <list>", "last element of <lilst>, "element <expr> in <list>"
  3. Equation definitions
    • Internal equations are completely defined with a body
    • External equations are identified with a complete signature but no body and may have an external reference URL
    • "Built-in" functions are specific to reasoner/translator pairs and using the reasoner-specific information a set of built-in function definitions are provided so that references are linked to the definitions
  4. Implied properties in type checking and translation
  5. General visual graphing interface with a GraphViz-based default representation

What's Changed in the Grammar

  1. Numbers can have units--here's the SADL unitted quantity meta-model in SADL syntax:

    UnittedQuantity is a class,
                described by ^value with values of type decimal,
                described by unit with values of type string.
    UnittedQuantity has impliedProperty ^value.

  2. RDF properties are supported
  3. A property can be specified to be owl:DatatypeProperty or owl:ObjectProperty without actually giving a range
  4. User-defined data types have a new syntax (not backward compatible)
  5. Queries can be named, can be executed by name
  6. Queries can specify that a graphical output should be created

Miscellaneous

  1. There are some new implicit OWL models provided (besides the "default values" model defaults.owl).
    • The SADL base model supporting Equations, Named Queries
    • The SADL list model supports lists (as illustrated above)
  2. The SADL implicit model includes the following:
    • Concept definitions to support unitted numbers
    • Concept definitions to support implied properties
    • The model is a SADL model so that it is extensible; projects can add their own implicit concepts
